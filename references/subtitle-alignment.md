# Subtitle Alignment

这是这个 skill 最容易返工的地方，必须按优先级处理。

## 优先级

1. 平台字幕 / 源字幕原始时间戳
2. 全局提前或延后
3. 单句 patch
4. 强制对齐 / 重新估时

如果任务进入最终成片或双语滚动字幕，先阅读并执行 [subtitle-stability.md](subtitle-stability.md)。本文件只负责时间校准原则，完整流程以稳定流程为准。

## 默认策略

- 中文字幕以可读性为先
- 英文字幕作为辅助层
- 允许字幕略早出现，不要整体偏晚
- 目标是“开口即见字”，不是“说完才弹字”
- 断句优先按语义，其次按长度
- 字幕底边距要先避开原视频信息层，再谈视觉居中
- 中英文必须同起同落，不要分别估时
- 滚动字幕允许短暂两个 cue 共存，但必须分轨显示，不能视觉重叠

## 全局 offset

当整段都慢了大约 0.3s 到 1.0s，优先使用全局偏移：

- 先试 0.3s
- 再试 0.5s
- 再试 0.8s
- 只有确认整体确实慢，才推到 1.0s 左右

每次调整都要写进 `render_manifest.json`。

建议把最终使用的参数写成：

```json
{
  "source_time_basis": "source.en.srt",
  "lead_sec": 1.2,
  "end_policy": "source_end",
  "single_cue_patches": []
}
```

## 局部 patch

如果只有少数句子不跟嘴：

- 先看该句是否本来就有长停顿
- 再调单句 start/end
- 不要把整段都重新切成更碎的块

## 不推荐的做法

- 把所有字幕重新估时
- 为了“滚动字幕”打乱原句时间
- 让字幕在说完之后才出现
- 过度切碎导致阅读负担增加
- 为了避免同屏两句而把下一句字幕整体推迟
- 在已经带字幕的视频上二次烧录

## 检查方法

- 抽取开头 10 秒看是否一开口就出现字幕
- 检查第一条字幕是否过晚
- 检查高密度段是否遮挡嘴型
- 检查字幕是否在镜头切换时仍可读
- 检查长句是否被拆成自然断句，而不是只按字符数硬切
