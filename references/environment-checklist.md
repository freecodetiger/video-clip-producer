# 环境检查清单

这不是装机清单，而是“当前任务能不能继续”的能力清单。

## 检查原则

- 优先复用用户已经可用的环境
- 先探测能力，再决定要不要补缺
- 只检查当前路径需要的能力，不把可选能力当硬门槛
- 不主动创建新的虚拟环境、conda 环境或容器，除非用户要求

## 分阶段检查

### 1) 通用

- 可用的 Agent 工具
- 可执行的 shell / 文件读写权限
- 输出目录可写

### 2) 摄取阶段

当前任务如果要下载或接管源文件，再检查：

- `python3`
- `yt-dlp`
- `ffmpeg`
- `ffprobe`

### 3) 字幕抓取阶段

只有需要从平台拉字幕时再检查：

- `--cookies-from-browser <browser>` 或 `cookies.txt`

### 4) 渲染阶段

只有当前渲染路径会用到时再检查：

- `ffmpeg` 支持 `subtitles`
- `ffmpeg` 编译时包含 `libass`
- `ffmpeg` 支持 `drawtext`（仅在确实使用文字叠加时）

### 5) 可选增强

下面这些是加分项，不是硬门槛：

- Python 可导入 `PIL`
- Python 可导入 `cv2`
- 本地 B-roll 目录可读
- 网络可访问免版权视频素材源

## 环境通过后的最低输出

只要当前任务所需能力齐了，skill 至少要能输出：

1. 下载与字幕摄取结果，或本地接管结果
2. 候选片段列表，或已确认片段的渲染计划
3. 排序分数，或渲染参数
4. 用户下一步提示

## 环境未通过时的输出

只报告：

- 缺少什么
- 哪些步骤还能继续
- 当前任务的最小补齐方式
- 不要为了一个缺口，建议整套重装环境

## B-roll 特例

本地 B-roll 目录缺失不是失败条件。默认策略是：

1. 先问用户是否已有本地 B-roll
2. 如果没有，自动从网络获取免版权自然素材
3. 如果网络素材也失败，再报告可继续的降级方案
